# 执行计划

## 概述

加载计划、严格审查、执行所有任务、完成时报告。

**开始时宣布：**"我正在使用执行计划技能来实现这个计划。"

**注意：**告诉你的合作伙伴，Superpowers 在有子代理支持的情况下效果更好。如果在子代理支持的平台上运行（如 Claude Code 或 Codex），其工作质量会显著提高。如果有子代理可用，请使用 superpowers:subagent-driven-development 而不是这个技能。

## 执行流程

### 步骤 1：加载并审查计划
1. 阅读计划文件
2. 严格审查 - 识别对计划的任何疑问或顾虑
3. 如果有顾虑：在开始前向合作伙伴提出
4. 如果没有顾虑：创建 TodoWrite 并继续

### 步骤 2：执行任务

对于每个任务：
1. 标记为进行中
2. 严格遵循每一步（计划包含小步骤）
3. 按规定运行验证
4. 标记为已完成

### 步骤 3：完成开发

所有任务完成并验证后：
- 宣布："我正在使用完成开发分支技能来完成这项工作。"
- **必需的子技能：** 使用 superpowers:finishing-a-development-branch
- 遵循该技能来验证测试、呈现选项、执行选择

## 何时停止并寻求帮助

**立即停止执行当：**
- 遇到阻碍（缺少依赖、测试失败、指令不清晰）
- 计划有关键性缺陷导致无法开始
- 不理解某条指令
- 验证反复失败

**寻求澄清而不是猜测。**

## 何时回溯前面的步骤

**返回审查（步骤 1）当：**
- 合作伙伴根据你的反馈更新了计划
- 基本方法需要重新思考

**不要强行突破阻碍** - 停下来并寻求帮助。

## 记住
- 首先严格审查计划
- 严格遵循计划步骤
- 不要跳过验证
- 当计划提到时引用技能
- 遇到阻碍时停止，不要猜测
- 未经用户明确同意，切勿在 main/master 分支上开始实施

## 集成

**必需的工作流技能：**
- **superpowers:using-git-worktrees** - 确保隔离的工作空间（创建或验证现有）
- **superpowers:writing-plans** - 创建此技能执行的计划
- **superpowers:finishing-a-development-branch** - 所有任务完成后完成开发
